# Chronos-V2: 4D 认知超图引擎 (Cognitive Hypergraph Engine)

> “不要预设世界，让 AI 去理解世界。”

**Chronos-V2** 是对传统时序数据库（TSDB）、静态知识图谱（GraphDB）以及向量检索（RAG）的一次底层概念合并。它是专门为大语言模型（如 Minimax M2.7、GPT-4o、Claude 3.5）设计的 **“即插即用型、自进化数字记忆外脑”**。

## 为什么说这次重写是真正“颠覆性”的？

在查阅了包括 Graphiti、Zep、Pathway 等前沿开源框架后，我摒弃了“预定义 Pydantic Schema（例如硬编码毛利率、净利润字段）”的做法，转而采用**大一统的架构模型**。

### 1. 万物皆 4D Tuple (S-P-O-T)
不再区分“实体属性”和“实体关系”。在这个系统中，所有信息都被拆解为四维元组：**Subject (主), Predicate (谓), Object (宾), Time (时)**。
- 财务指标：`("宁德时代", "毛利率", "28.5%", "2023-09")`
- 投资关系：`("宁德时代", "投资", "固态电池", "2023-10")`
- 宏观事件：`("美联储", "宣布", "降息", "2024-09")`
这使得我们只需要一个极简的存储结构，就能表达金融市场的全部动态，并且支持 O(1) 的多维交叉索引查询。

### 2. 完全 Schema-less，把权力交给大模型
在 `chronos_llm.py` 中，我通过 OpenAI 兼容 API 接入你的大模型（如 Minimax）。系统**不定义任何属性名称**，大模型阅读文本后，自由“发明”并提取出最能概括商业逻辑的 Predicate（谓语）。
这意味着：如果今天出了一个新概念叫“低空经济市占率”，系统无需任何修改，图谱里就会自动生长出这条时间线。

### 3. 认知巩固机制 (The "Sleep" Mechanism)
自由发散会导致图谱里充满噪音（如 "宁王" 和 "CATL" 同时存在，"毛利" 和 "毛利率" 冲突）。
在 `chronos_sleep.py` 中，我引入了类似于人类睡眠时整理记忆的“认知巩固”机制。系统在后台会定期调用 LLM（或聚类算法）对图谱进行反思，自动合并同义词、归一化关系，从而实现**图谱本体（Ontology）的自我进化与纠错**。

## 极简的 4 个核心文件

1. **`chronos_graph.py`**: 万能 4D 记忆引擎核心，取代传统图数据库。纯 Python 字典实现的超高并发倒排索引。
2. **`chronos_llm.py`**: 基于 OpenAI 兼容协议（支持 Minimax）的通用抽取管道。
3. **`chronos_sleep.py`**: 睡眠与记忆整理机制（图谱自进化）。
4. **`chronos_mcp.py`**: 极简的网关。大模型只需调用一个万能的 `query_graph` 函数，即可查询任意维度的关系和趋势。

## 如何运行演示？

```bash
# 核心架构仅依赖 openai 官方包
pip install openai

# 运行惊艳的测试用例（包括记忆注入、噪音查询失败、系统睡眠自愈、再次精准查询）
python chronos_mcp.py
```
